# Engineering Memo: v0.2 Plan

Version: v0.2  
Date: 2026-05-14  
Author: Codex  
Area: Planning

## 1. Summary

This memo defines the v0.2 execution plan for completing deferred v0.1 items without changing AirPath's product category, positioning, or governing scope.

## 2. Context

v0.1 final handoff documented the following deferred items:

```txt
report screenshot embedding
direct 3D drag
undo / redo
shader heatmap
warning clustering
bundle-size optimization
CI / deployment
```

The v0.2 goal is to implement these items where practical and document any accepted limitations with specific reasons.

## 3. Execution Order

1. Report screenshot embedding
   - Add viewport canvas capture with WebGL preserve buffer.
   - Capture layout, thermal, and airflow views before report generation.
   - Embed captured images into the existing report sections.
   - Extend Playwright acceptance to assert report images exist.

2. Direct 3D drag
   - Add floor-plane style rack dragging in the 3D viewport.
   - Keep inspector move buttons as fallback.
   - Ensure drag updates scenario data and preview simulation output.

3. Undo / redo
   - Add bounded history snapshots around scenario-mutating actions.
   - Add buttons and keyboard shortcuts for undo/redo.
   - Ensure undo/redo restores scenario and simulation preview.

4. Shader heatmap
   - Replace the mesh-cell thermal slice with a shader-driven plane using the solver temperature field as a data texture.
   - Preserve the approved cyan/slate/amber/orange/red thermal scale.
   - Keep heatmap driven by simulation data.

5. Warning clustering
   - Group nearby or repeated warning categories in the viewport.
   - Keep detailed warning cards in the right panel.
   - Ensure clustered pins remain clickable and focus related warnings.

6. Bundle-size optimization
   - Add Vite manual chunk splitting for React, Three, and AirPath package layers.
   - Keep build behavior unchanged.

7. CI / deployment
   - Add GitHub Actions for install, lint, test, build, and Playwright acceptance.
   - Add Vercel configuration for `apps/web/dist`.
   - Add README notes only where strictly necessary for running/deploying the implemented app.

## 4. Risks

| Risk | Impact | Mitigation |
|---|---|---|
| WebGL canvas capture may be blank without preserve buffer | Report screenshots fail | Enable `preserveDrawingBuffer`, test iframe images in Playwright |
| Automated view switching for report capture may race rendering | Wrong or duplicate screenshots | Wait for animation frames between mode changes |
| Direct drag may conflict with orbit controls | Poor viewport interaction | Disable orbit while dragging and retain inspector fallback |
| Undo snapshots can become heavy | Memory growth | Use bounded history and scenario-only snapshots |
| Shader heatmap data texture may break SSR/build types | Build failure | Keep implementation local to Three viewport and covered by build |
| Warning clustering could hide severity | Risk visibility regression | Cluster count and max severity shown; full list remains in Warnings tab |
| CI with Playwright may be slow | CI timeout | Use Chromium install and existing single acceptance flow |
